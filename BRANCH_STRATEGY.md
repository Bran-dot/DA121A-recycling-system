# Branch Strategy – Solo Work

## Repository
- Name: DA121A-recycling-system
- Owner: Bran-dot (solo)
- Visibility: Public

## Branch Rules
- `main` branch is protected
- No direct commits to `main`
- All work on feature branches named: `feature/[description]`
- Pull request required for every merge
- I approve my own pull requests (solo workflow)

## Branch naming examples
- `feature/readme-setup`
- `feature/requirements`
- `feature/domain-concepts`
- `feature/uml-sketches`
- `feature/strategy-pattern`
- `feature/layered-architecture`
- `feature/junit-tests`

## Workflow
1. Create feature branch from `main`
2. Make small, focused commits
3. Push branch to GitHub
4. Open Pull Request to `main`
5. Self-review and merge
6. Delete feature branch (optional)
